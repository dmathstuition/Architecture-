Phase 1 — Foundation

System Architecture

Every layer of the SaaS platform explained — from browser to database

🖥️

Frontend Layer

The user-facing app — public church website plus the protected admin dashboard. Built for speed, SEO, and a clean feel that non-technical staff can actually use.

Next.js 14 App Router Tailwind CSS ShadCN UI React Query Framer Motion
⚙️

Backend Layer

API routes inside Next.js handle all server logic — authentication, data reads/writes, file upload processing, and payment webhook handling. Zero separate server needed.

Next.js API Routes Node.js Runtime Prisma ORM JWT / NextAuth Zod Validation
🗄️

Database Layer

PostgreSQL on Neon cloud stores all relational data. Prisma gives type-safe querying. Every record carries a tenantId for full church data isolation.

PostgreSQL Prisma ORM Neon Cloud DB Multi-tenant Schema
☁️

Media & Streaming

Cloudinary handles every video and audio sermon upload, auto-generates thumbnails, and delivers everything via global CDN — no separate media server required.

Cloudinary CDN Video Upload API Audio Streaming Auto Thumbnails
